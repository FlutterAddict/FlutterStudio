Introduction

There are two main overarching themes from writing consistent, robust, fast code.

Consistency

Be consistent with your code.

When it comes to things like formatting and casing, arguments about which one is better are subjective and impossible to resolve.

What we do know, is that being consistent is objectively helpful.

If two pieces of code look different, it should be because they are different in some meaningful way.

When a bit of code stands out and catches your eye, it should do for a useful reason.

Briefness

Be brief with your code.

Dart was designed to be familiar, so it inherits many of the same statements and expressions as C, Java, JavaScript and other languages.

But Dart was created because there is a lot of room to improve on what those languages offer.

If there are many ways to say something, you should generally pick the most concise one.

That doesn't mean you should cram your whole program into single line. The goal is to be economical, not dense.

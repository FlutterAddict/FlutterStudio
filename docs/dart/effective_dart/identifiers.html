Dart Style Guide: Identifiers

Three Flavors of Identifiers

Identifiers in Dart come in three flavors:

  1. UpperCamelCase - first letter of each word is capitalized, including first word

  2. lowerCamelCase - first letter of each word is capitalized except of the first word which is always lowercase, even if it's an acronym

  3. lowercase_with_underscores - words in lowercase only, even for acronyms, separated with undescores (_)

Use UpperCamelCase

Name types using UpperCamelCase - classes, enums, typedefs, and type parameters should capitalize first letter of each word including the first word and use no separators.

// good
class SliderMenu { ... }
class HttpRequest { ... }
typedef num Adder(num x, num y);
typedef Predicate<T> = bool Function(T value);

Linter rule: camel_case_types

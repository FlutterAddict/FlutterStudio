Important Concepts

As you learn about the Dart language, keep these facts and concepts in mind.

I. Everything is an Object

Everything you can place in a variable is an object, and every object is an instance of a class.

Even numbers, functions and null are objects.

All objects inherit from the Object class

II. Dart Can Infer Types

Although Dart is strongly typed, type annotations are optional because Dart can infer types.

When you want to explicitly say that no type is expected (if an object isn't restricted to a single type), use the special type dynamic or Object type.

Let me demonstrate this with examples:

Type Inference Example

var n = 42;

In the code above, n is inferred to be of type int. n is no longer a var but it's an integer (int).

That means that if you try to assign a string to it:

n = 'esskeetit';

It will throw an error, because n is of int type and can't hold a String.

Dynamic Type Example

dynamic x = 42;
x = 'forty two';

This will work properly, without any error.

III. Generic Types Support

Dart also supports generic types, like List<int> (a list of integers) or List<dynamic> (a list of objects of any type).

IV. Variables

Dart supports:

  • top-level variables,

  • variables tied to classes (static variables),

  • variables tied to an object (instance variables, sometimes known as fields or properties).

V. Functions

Dart supports:

  • top-level functions, such as main(),

  • functions tied to a class (static methods),

  • functions tied to an object (instance methods),

  • functions nested within functions (nested/local functions).

VI. Libraries and Visibility

Unlike Java, Dart doesn't have public, private and protected keywords.

If an identifier starts with an underscore, e.g. var _privateVar; it is private to its library.

See Libraries and Visibility for details.

VII. Identifiers / Naming

Identifiers (the names for your variables, functions, classes etc...) can start with a letter or underscore, followed by any combination of those characters and digits.

var _privateIdentifier123;
var validVariable2Name;
var iamc0rrect_t00;

Note: Identifiers starting with an underscore (_) will be private to its library.

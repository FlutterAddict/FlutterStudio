Using Material Components

Flutter provides a number of widgets that help you build apps that follow Material Design.

Material Apps

Using the MaterialApp is entirely optional but a good practice.

It will build a number of useful widgets at the root of your app, including Navigator which lets you transition smoothly between screens of your app.

Many widgets need to be inside of a Material App to display properly, in order to inherit theme data.

Example

Here is a code for a very basic Material app.

import 'package:dart/material.dart';

void main() {
  runApp(
    MaterialApp(
      title: 'Your App Title',
      home: Scaffold( // Scaffold is a layout for the major Material Components
        appBar: AppBar(
          leading: IconButton(
            icon: Icon(Icons.menu),
            tooltip: 'Open Navigation Menu',
            onPressed: null,
          ),
          title: Text('Home'),
          actions: <Widget>[
            IconButton(
              icon: Icon(Icons.search),
              tooltip: 'Search',
              onPressed: null,
            ),
          ],
        ),
        body: Center(
          child: Text('Material Hello World'),
        ),
        floatingActionButton: FloatingActionButton(
          tooltip: 'Example FAB',
          child: Icon(Icons.add),
          onPressed: null,
        )
      ),
    ),
  );
}
